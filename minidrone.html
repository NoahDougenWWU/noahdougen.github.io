Drone Object Tracking with Aruco Tags

Overview

A team of four people worked together to create object tracking software for the Crazyflie 2.1 minidrone. Our software uses Aruco tags to track the positions of the drone and a target object. The drone can then be controlled by moving the target object.

Key Takeaways

  • Object tracking and state estimation
  • Experience with libraries like OpenCV
  • Collaborative project experience

Details

Our team setup a system that allowed a Crazyflie drone to be controlled using two Aruco tags, one attahced to the drone and one attached to a target object. The system involved a camera scanning for the two Aruco tags, estimating both their positions, and instructing the drone to remain close to the target object with PID controllers. The system mainly utilizes OpenCV's Aruco tag library for position estimation.

Sense-Think-Act Loop

The system starts with the stationary camera scanning for the two Aruco tags and the drone taking off and hovering. The states (xyz positions) are estimated using known information about the tag size and camera. To create accurate estimates of the drone and target's states the ten most recent states are averaged together. This reduces error from the occasional misread caused by glare.

With positions estimated, the drone can be commanded to move up/down, backward/forward, and left/right. To control this, we developed three PID controllers, one for each xyz coordinate. An offset is used to keep drone slightly lower than the target so both Aruco tags can be read by the camera. The action commands are given in real-time once every tenth of a second.

Kalman Filtering

While it was unecessary to implement into the final system, I also developed a kalman filtering algorithm for estimating the position, rotation, and velocity of an Aruco tag. This would be ideal for situations where the tag became obscured for a few seconds as it would aloow the system to predict the tag's state. Below is a GIF demonstrating the algorithm.
